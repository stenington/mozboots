Instructions

By and large, interacting with the app and viewing the source will be the best way to understand it. However, additional notes are provided below to help provide context for the project.

Login

Login is handled with the Persona Observer API.

The app lets Persona manage user state between page refreshes and displays a wait message while synchronization happens. Because routing should otherwise be handled by the client application, this screen should really only be seen by a user when they first visit the app.

To see this screen in action, refresh any page. To force a delay add ?personaLagMs=3000 or some other millisecond value to the url.

Route handling

Routes are handled by Backbone.Router using HTML5 pushState support where available. The client app is served from most urls without any server-side rendering, at this point. This has several drawbacks, and alternatives can be explored in future iterations of this sandbox.

Templates

Templating uses precompiled nunjucks templates. clientapp/templates.js is a thin wrapper over the nunjucks API to conform to the convention set forth in Human Javascript.
